# ChocolatesByNumbers — Mathematical explanation

Problem: Given N chocolates placed in a circle and a step size M, start at position 0 and repeatedly move M positions clockwise; how many distinct chocolates will be eaten before returning to a previously-eaten chocolate?

Key observation

Positions visited form an arithmetic progression modulo N. The cycle length equals the order of M in \(\mathbb{Z}/N\mathbb{Z}\), more concretely the number of distinct positions visited is:

\[\text{count} = \frac{N}{\gcd(N,M)}.\]

Proof sketch

Let g = gcd(N,M). Then M = g M', N = g N' with gcd(N',M')=1. Stepping by M modulo N cycles through the residue classes congruent modulo g. The subgroup generated by M has size N' in the quotient modulo g, so there are N' distinct residues per congruence class; since we start at 0 which lies in one class, we visit exactly N' = N/g distinct positions.

Algorithmic implication

Compute the gcd and return N/g. Complexity: O(log(min(N,M))) using Euclid's algorithm.

Notes

This short, exact closed-form is why the coding solution for the classic task is simply gcd-based and optimal.
